状態フラグは、投票受付中・結果反映待ち・結果反映済みを区別するための進行管理用ラベルです。本章では意味だけを定義し、遷移制御は第3章へ送ります。

ID区分内容主体・責任関連語句備考

1-7　データ保存・進行更新境界定義

保存と進行更新を混同しないための最重要境界です。

保存は記録、進行更新は作戦状態の変化です。この2つを分けることで、誤って先に状態を進める事故を防ぎます。

ID区分内容主体・責任関連語句備考
1-7データ保存・進行更新境界game-state.json に現在地、パラメーター、選択肢、内部評価、状態フラグを永続化する。作戦状態を進める進行更新は applyChoiceResult 実行時の1回のみ行う。選択肢作成やプロンプト作成時に保存する場合でも、進行結果は書き込まない。Railway Actions / Backendgame-state.json、applyChoiceResult、保存、進行更新保存と進行更新を分離する。詳細な保存設計は第3章で扱う。

1-7補足　保存と進行更新の分離

区分内容例備考
保存メモ、投稿内容、選択肢案、採用行動、状態フラグ、内部情報などを記録する処理投稿内容保存、投票状況メモ、採用行動メモ保存のみでは作戦状態は進まない。
進行更新作戦状態、パラメーター、危険度、ルート進行などを実際に変更する処理applyChoiceResult による結果反映採用行動確定後に1回のみ行う。
